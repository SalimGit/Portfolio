RAPPORT DE STAGE

Dans le cadre de ma formation en BTS Services Informatiques aux Organisations. J’ai effectué un stage pour Iboski durant dix semaines, du 25 novembre 2019 au 3 février 2020 dont le but était la création d'un site de livraison de repas.

IBOSKI

IBOSKI est une société qui accompagne les porteurs de projets dans la création et le développement de leurs activités en proposant des solutions digitales. La société se complète pour former une équipe de 3 passionnés aux parcours très différents qui regroupent leurs expertises dans le but d'accompagner chaque idée jusqu'à sa réalisation et la satisfaction de la clientèle. IBOSKI aide les entreprises à soigner leurs identité visuelle et leurs présence sur le web dans le but d'atteindre divers objectifs: fidélisation, croissance du taux de conversion, trafic, publicité ciblée, image de marque référencement. OKmanger et Skyplus font partie de ses sites web.

MISSION

Recruté en tant que développeur web pour IBOSKI plusieurs missions m’ont été confiées. Nous avions discuté, du cadre dans lequel j’allais évoluer et de ce dont il avait besoin dans le but de la réalisation du site.

Front-End

La première mission que j'ai effectuer lors de mon stage était du front-end il s'agit des éléments du site que l'on voit à l'écran et avec lesquels on peut interagir. Cette mission c'est décomposé en plusieurs étapes.

Navbar #1
L'objectif était de réaliser une barre de navigation pour que l'utilisateur pour parcourir plus facilement les différentes pages du site.
Modal de connexion #2
Le but était de créer un modal qui s'ouvre lorsque l'on appuie sur le bouton "Connexion/inscription" pour permettre au client de réaliser son souhait.
Barre de recherche #3
Lors de cette tache l'objectif était de réaliser une barre de recherche pour que l'utilisateur puisse chercher directement le restaurant de son choix ou les restaurants d'une ville.
Card accueil #4
Le but de cette tache était la création de "Card" dans la page d'accueil pour donné envie au client d'aller plus dans le site.
Footer #5
L'objectif était la réalisation d'un pied de page qui contient des informations utiles à l'utilisateur comme les mentionts légales, les contacts ...
Carrousel #6
Le but de cette tache était la réalisation d'un "Carrousel" qui propose des restaurants au client selon ses précédentes commande
Card LesRestaurants #7
Lors de cette tache le but était de créé des "Cards" pour y placer les différents restaurants partenaire.
Card Restaurant #8
Lors de cette tache le but était de créé des "Cards" pour y placer les menus des différents restaurants partenaire.
Panier #9
L'objectif était de réaliser un "Panier" pour que le client puisse suivre sa commande et par la suite la payer.
Payement #10
Le but était de réaliser une page pour que le client puisse payer ce qu'il a précédemment mis dans son panier.

Back-End

La deuxième mission que j'ai effectuer lors de mon stage était du back-end ce terme désigne un étage de sortie d'un logiciel devant produire un résultat. On l'oppose au front-end qui lui est la partie visible de l'iceberg. Cette mission c'est décomposé en plusieurs étapes.

Inscription/Connexion #1
Le but de cette mission était de réaliser la phase d'inscription et de connexion, il fallait que lorsqu'un utilisateur s'inscrit l'information soit bien transmise et enregistré et par la suite réutilisé pour pouvoir se connecter.
Card LesRestaurants #2
Le but de la page LesRestaurant est d'afficher les restaurants partenaire selon certains critères (lieu, horaire d'ouverture...) il fallait donc prendre les informations de la base de donné pour ensuite l'afficher au client selon les critères choisis.
Card Restaurant #3
L'objectif de cette page est d'afficher le menu d'un restaurant en particulier le but était donc de récupérer ces informations pour les transmettre au client.
Panier #4
L'objectif de cette tache était prendre en compte la commande du client et par la suite réaliser le moyen de payement (notamment avec Stripe)
Envoie email #5
Pour cette tache le but était que lors de la phase d'inscription un mail de confirmation soit envoyé à l'adresse mail indiquer.

Outil

Outils utilisé pour la réalistion du projet.

PHP

Principalement utilisé pour produire des pages Web dynamiques, PHP est un langage impératif orienté objet.

Laravel

C'est un framework web open-source écrit en PHP respectant le principe modèle-vue-contrôleur.

Bulma

Bulma est un framework CSS.

Visual Studio Code

Visual Studio Code est un éditeur de code extensible développé par Microsoft pour Windows, Linux et macOS (IDE).

Trello

Trello est un outil de gestion de projet, il repose sur une organisation des projets en planches représentant des tâches.

Discord

Discord est un logiciel conçu pour communiquer.

GitHub

GitHub est un service web d'hébergement et de gestion de développement de logiciels.

Javascript

JavaScript est un langage de programmation de scripts principalement employé dans les pages web interactives.
